Can you think of an anology to describe HTML, CSS and DOM? How would you describe it to your non-techinical friend?

I think of HTML and CSS as layers of style lying on top of the html structure. And DOM as a viewing screen you look through to see those layers and seeing how the document is structured like a tree.

What is meant by boxifying design?

Boxes, boxes, boxes. Drawing boxes around visual elements in the design so you can rebuild it as a web design. Basically breaking down the elements on the web page design so you can re-create it. Those boxes help you build it by breaking down the design so it's easier to code and visualize.

What is the box model?

The Box Model - basically breaking down more the boxify design and putting structure in the boxes in a way to achieve the ideal design or final product of the web design.
